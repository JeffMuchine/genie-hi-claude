## Bugs Found & Fixed

### ISSUE-001 — Login: No client-side email validation [HIGH]
- **Page:** `/login`
- **Symptom:** Submitting blank email or malformed address (`foo@`) would call Firebase directly, causing a cryptic API error rather than a friendly message.
- **Fix:** Added regex validation in `LoginPage.jsx` before Firebase call.
- **Evidence:** Code at `frontend/src/pages/LoginPage.jsx:39`
- **Status:** ✅ FIXED

### ISSUE-002 — GET /resume returns 404 when no resume stored [HIGH]
- **Page:** `/app` (on mount)
- **Symptom:** `MainApp.jsx` calls `GET /resume` on load. When no resume was stored, the backend returned 404, causing a noisy console error on every fresh session.
- **Fix:** Changed backend to return `200 {"exists": false}` instead of 404.
- **Evidence:** `backend/routers/resume.py:77-79`, verified via `curl http://localhost:8000/resume`
- **Status:** ✅ FIXED

### ISSUE-003 — Claude API 401 retried 3 times with 7s delay [MEDIUM]
- **Page:** Job parsing / document generation
- **Symptom:** Invalid Anthropic API key caused 3 retry attempts with 1s/2s/4s backoff (7s total wasted delay) before failing.
- **Fix:** Added fast-fail on auth errors (`401`, `authentication_error`, `invalid x-api-key`) in `claude_service.py`.
- **Evidence:** `backend/services/claude_service.py` retry loop
- **Status:** ✅ FIXED

### ISSUE-004 — React Router v6 future flag warnings on every load [MEDIUM]
- **Page:** All pages
- **Symptom:** Two console warnings on every page load about `v7_startTransition` and `v7_relativeSplatPath`.
- **Fix:** Added `future={{ v7_startTransition: true, v7_relativeSplatPath: true }}` to `<BrowserRouter>` in `App.jsx`.
- **Evidence:** `frontend/src/App.jsx:149`, verified clean console after fix
- **Status:** ✅ FIXED

### ISSUE-005 — Upload Resume button aria-label ambiguous [MEDIUM]
- **Page:** Resume upload modal
- **Symptom:** Browse tool selector `@e` for "Upload Resume" was ambiguous — matched both the modal confirmation button and the Step 1 "+ Upload resume" text button.
- **Fix:** Added `aria-label="Confirm and upload resume file"` to the modal button.
- **Evidence:** `frontend/src/components/ResumeUploadModal.jsx:389`
- **Status:** ✅ FIXED
